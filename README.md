# Credit Card Fraud Detection

A complete machine learning pipeline for detecting fraudulent credit card transactions on highly imbalanced data (~0.17% fraud). Built as an AI engineer interview practice project.

## Key Topics Covered

- **Imbalanced classification** — SMOTE oversampling, class weights, why accuracy is useless
- **Precision vs Recall tradeoff** — the business cost of false negatives vs false positives
- **Threshold tuning** — finding optimal decision thresholds for different objectives
- **ROC vs PR curves** — why AUPRC is the right metric for imbalanced problems

## Project Structure

```

├── notebooks/

```

## Setup

```bash
pip install -r requirements.txt
```

Download the dataset from [Kaggle](https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud) and place `creditcard.csv` in the project root.

## Usage

### 1. Preprocess

```bash
python src/preprocess.py --data-path creditcard.csv --output-dir data
```

### 2. Train

```bash
# Train all models and compare
python src/train.py --data-dir data --model-dir models --train-all

# Or train a specific model
python src/train.py --data-dir data --model-dir models --model XGBoost
```

### 3. Evaluate

```bash
python src/evaluate.py --data-dir data --model-path models/XGBoost.joblib --output-dir models
```

### 4. Inference

```bash
python src/inference.py \
    --model-path models/XGBoost.joblib \
    --scaler-path data/scaler.joblib \
    --input-csv new_transactions.csv \
    --threshold 0.3
```

### 5. Serve the API

```bash
# Start the FastAPI server
uvicorn src.api:app --reload --host 0.0.0.0 --port 8000
```

The interactive docs are available at `http://localhost:8000/docs` (Swagger UI) and `http://localhost:8000/redoc`.

#### API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/health` | Health check & model status |
| `GET` | `/model/info` | Model metadata, threshold, features |
| `POST` | `/predict` | Predict fraud for a single transaction |
| `POST` | `/predict/batch` | Predict fraud for multiple transactions |
| `PUT` | `/threshold` | Update decision threshold at runtime |

| Variable | Default | Description |
|----------|---------|-------------|
| `MODEL_PATH` | `models/XGBoost.joblib` | Path to the trained model |

## Dataset

The [Credit Card Fraud Detection](https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud) dataset contains 284,807 European credit card transactions from September 2013. Features V1–V28 are PCA-transformed for confidentiality. Only `Time` and `Amount` are original features.

## Models

| Model | AUPRC | AUROC | F1 |
|-------|-------|-------|----|
| Logistic Regression | ~0.70 | ~0.97 | ~0.73 |
| Random Forest | ~0.82 | ~0.98 | ~0.85 |
| XGBoost | ~0.84 | ~0.99 | ~0.87 |

*Scores are approximate — run the pipeline to get exact numbers.*
